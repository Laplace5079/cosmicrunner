# Mobile Testing Guide

## 已完成的移动端适配改进

### 1. 响应式游戏元素
- ✅ 玩家大小根据屏幕尺寸自动缩放 (40-60px)
- ✅ 玩家位置固定在屏幕左侧 15%
- ✅ 重力和跳跃力度根据屏幕高度缩放
- ✅ 地面位置在屏幕底部 15%
- ✅ 障碍物和星星生成位置考虑屏幕宽度

### 2. 触屏控制优化
- ✅ 整个屏幕可作为触控区域
- ✅ 点击上半部分 = 跳跃
- ✅ 点击下半部分 = 滑行
- ✅ 支持滑动手势 (上滑/下滑)
- ✅ 防止默认触摸行为 (滚动、缩放)
- ✅ 触摸反馈优化

### 3. UI 安全区域
- ✅ HUD 使用 `env(safe-area-inset-top)` 避开刘海屏
- ✅ 声音按钮定位在安全区域
- ✅ 所有按钮最小尺寸 44x44px (iOS 触控标准)
- ✅ 暂停按钮有防误触设计

### 4. 响应式布局
- ✅ 字体使用 `clamp()` 自适应
- ✅ 按钮大小根据屏幕调整
- ✅ 小屏幕 (< 400px) 优化
- ✅ 横屏模式特殊布局
- ✅ 统计面板网格自适应

### 5. 性能优化
- ✅ 背景星星数量根据屏幕宽度调整
- ✅ 碰撞检测 hitbox 缩放
- ✅ 粒子效果缩放
- ✅ 60 FPS 目标帧率

### 6. Poki SDK 集成
- ✅ Poki SDK 脚本已加载
- ✅ 游戏开始/停止事件已集成
- ✅ 商业化功能已准备

## 如何在手机上测试

### 方法 1: 本地服务器
```bash
cd poki-game
./start-server.sh
```

然后在手机浏览器输入电脑的 IP 地址:
```
http://[你的 IP 地址]:8888
```

### 方法 2: 使用 GitHub Pages
1. 在 GitHub 仓库设置中启用 GitHub Pages
2. 选择 main branch 作为源
3. 访问: `https://Laplace5079.github.io/cosmicrunner/`

### 方法 3: 使用在线服务
上传到以下任一平台:
- Netlify (拖拽上传)
- Vercel
- Surge

## 调试技巧

### iOS Safari 远程调试
1. iPhone 设置 → Safari → 高级 → 网页检查器 (开启)
2. Mac Safari → 开发 → 选择你的设备
3. 打开控制台查看错误

### Chrome 远程调试
1. Chrome 浏览器访问 `chrome://inspect`
2. 连接 Android 设备
3. 查看和调试页面
